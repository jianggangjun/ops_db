## 模块状态

### ✅ install — MySQL 安装

**功能**：自动化安装 MySQL + xtrabackup，支持本地和远程（SSH）

**已实现**：
- OS 自动探测（CentOS/Ubuntu/Debian/Rocky/AlmaLinux）
- 交互式 / 非交互式安装
- 多版本支持（5.7 / 8.0 / 8.4 / 9.0）
- 多角色（single / master / slave）
- 镜像源选择（tencent / aliyun / tsinghua / official / intranet）
- **内网源支持（`--mirror=intranet`，通过环境变量配置）**
- 前置检查（磁盘空间 / 依赖包 / 端口占用）
- 远程安装（通过 SSH stdin tarball 推送）
- 随机 root 密码生成

**待完善**：
- 安装后首次登录强制修改密码流程
- 离线安装支持（本地 rpm/deb 包）

---

### ✅ backup — 备份

**功能**：全量 / 增量 / 逻辑备份

**已实现**：
- **Mode B 链式增量备份**：每个增量基于上一次增量（而非每次基于全量），LSN 链完整
- 全量备份（xtrabackup --backup）
- 增量备份（xtrabackup --backup --incremental-basedir=<最新备份>）
- 逻辑备份（mysqldump --single-transaction --master-data=2）
- 备份元数据记录（binlog position / GTID / LSN）
- 过期清理（--expire-days）
- 并行压缩（--compress）
- **备份加密（--encrypt / --encrypt-key-file）**
- **备份历史 SQLite 数据库（~/.ops_db/backup_history.db）**
- 远程备份（--ssh-host）
- 前置检查（磁盘空间 / 依赖 / 权限）

**待完善**：
- 备份验证（xtrabackup --export --dry-run）

---

### ✅ restore — 恢复

**功能**：全量 / PITR / PITR链 / partial / binlog-replay 恢复

**已实现**：
- **PITR 链式恢复（pitr-chain）**：按 LSN 顺序 apply 全量 + 多个增量，最后一次 prepare
- 全量恢复（xtrabackup --copy-back）
- **解密恢复（--decrypt-key-file）**：自动检测 .xbc 加密文件并解密
- PITR 时间点恢复（xtrabackup --prepare --binlog-dir --stop-datetime）
- Partial 单库恢复（discard → cp .ibd → import）
- Binlog replay 恢复（mysqlbinlog 定点恢复）
- Safety backup（恢复前自动备份现有数据）
- Dry-run 模式（binlog-replay 只生成 SQL 不执行）

**待完善**：
- 恢复后数据校验
- 恢复进度显示

---

### ✅ replicate — 主从配置

**功能**：全自动主从配置，支持远程自动安装备库

**已实现**：
- **远程自动安装**：如果备库未安装，自动 SSH 过去安装 MySQL
- 自动检测并修复 server-id 冲突
- 自动配置 bind-address（0.0.0.0）
- 自动创建 repl@% 复制账户（mysql_native_password 避免认证问题）
- 自动获取主库 binlog 坐标（FLUSH TABLES WITH READ LOCK + SHOW MASTER STATUS）
- 自动 CHANGE MASTER TO + START SLAVE
- 复制状态验证（Slave_IO_Running / Slave_SQL_Running / Seconds_Behind_Master）
- 数据覆盖风险警告 + --yes 跳过确认

**待完善**：
- GTID 模式自动化配置
- 半同步复制支持

---

### ⚠️ rebuild — 备库重搭

**功能**：备库重搭（延时大 / 故障 / 新机器）

**已实现**：
- 延时检测（SHOW SLAVE STATUS Seconds_Behind_Master）
- 基于位置重建（直接从当前复制点继续，不重建）
- 备库故障时全量重搭流程

**待完善**：
- 新机器重搭完整流程（远程安装 → 备份 → 恢复 → 建复制）
- 主库 binlog 已被 purge 时的处理（GTID 降级）

### ⚠️ check — 健康检查

**功能**：数据库健康检查

**已实现**：
- 连接性检查
- 版本检查
- 复制状态检查（IO/SQL 线程 / 延时 / 错误）
- 慢查询检查
- 连接数使用率
- root 密码安全检查
- 远程检查（--ssh-host）

**待完善**：
- 锁等待检测
- 磁盘空间检查
- 备份健康检查（最近一次全量距今是否超过 7 天）

---

### ✅ schedule — 定时备份调度

**功能**：通过 crontab 管理本地/远程 MySQL 的定时备份

**已实现**：
- schedule add：添加 cron job（本地或远程）
- schedule list：查看 ops_db 相关的调度列表
- schedule remove：按名称删除调度
- marker 机制（ops_db_backup_<name>）防止冲突
- SSH 远程模式（在远程机器上操作 crontab）

**待完善**：
- 调度任务执行日志记录
- 调度失败告警
- 调度执行结果记录到备份历史

```

✅ ops_db replicate # 主从配置
⚠️ ops_db rebuild   # 备库重搭
⚠️ ops_db check     # 健康检查
✅ ops_db schedule  # 定时调度
```
