# 离线安装功能设计

## 1. 背景与目标

### 痛点
- 内网环境无法访问外网镜像
- 网络不稳定（弱网）环境安装 MySQL/XtraBackup 经常卡在下载环节
- 外网镜像下载速度慢（腾讯云、阿里云在弱网下不稳定）
- MySQL + XtraBackup tarball 体积大（2-3 GB），不符合 CLI 工具定位

### 目标
为 ops_db 增加离线安装能力，支持两种模式：
- **内网镜像源模式**（推荐）：配置公司内网镜像源地址，走内网源安装，包体积 0 KB
- **离线包模式**（备选）：目标环境完全没有源时的终极方案

---

## 2. 安装模式

### 模式一：内网镜像源（推荐）

适用场景：内网有公司级镜像源（Nexus/JFrog/Artifactory）或云厂商内网镜像

```
公司内网 Nexus：   http://nexus.internal/repository/mysql/
腾讯云内网 apt：   https://mirrors.cloud.tencent.com/mysql/apt/
腾讯云内网 yum：   https://mirrors.cloud.tencent.com/mysql/yum/
阿里云内网 yum：   https://mirrors.aliyun.com/mysql/yum/
清华内网镜像：     https://mirrors.tuna.tsinghua.edu.cn/mysql/apt/
```

配置方式：
```bash
# 设置内网源环境变量（可写入 ~/.bashrc 持久化）
export INTRANET_MYSQL_YUM_REPO="http://nexus.internal/repository/mysql/yum/mysql-80-community-release-el8-4.noarch.rpm"
export INTRANET_MYSQL_YUM_REPO_EL8="http://nexus.internal/repository/mysql/yum/mysql-80-community-release-el8-4.noarch.rpm"
export INTRANET_PERCONA_YUM_REPO="http://nexus.internal/repository/percona/yum/release/latest/percona-release-latest.noarch.rpm"
export INTRANET_MYSQL_APT_REPO="http://nexus.internal/repository/mysql/apt/"
export INTRANET_PERCONA_APT_REPO="http://nexus.internal/repository/percona/apt/"

# 使用内网源安装
ops_db install --mirror=intranet --version=8.0
```

### 模式二：离线包（备选）

适用场景：目标环境完全没有源（如纯离线机），万不得已的终极方案

```
ops_db packages download --all
scp -r packages/ root@target:/tmp/
ops_db install --offline=/tmp/packages --ssh-host=192.168.56.8
```

## 2. 包目录结构

### 2.1 目录布局

```
ops_db/
├── packages/                          # 离线安装包目录
│   ├── mysql/
│   │   ├── mysql-5.7.44-linux-glibc2.12-x86_64.tar.gz
│   │   ├── mysql-8.0.41-linux-glibc2.28-x86_64.tar.gz
│   │   ├── mysql-8.4.3-linux-glibc2.28-x86_64.tar.gz
│   │   └── mysql-9.0.0-linux-glibc2.28-x86_64.tar.gz
│   ├── xtrabackup/
│   │   ├── percona-xtrabackup-2.4.28-linux-x86_64.tar.gz
│   │   ├── percona-xtrabackup-8.0.36-30.1-linux-x86_64.tar.gz
│   │   └── percona-xtrabackup-8.2.0-1-linux-x86_64.tar.gz
│   └── manifest.json                 # 包索引清单
```

### 2.2 命名规范

**MySQL tarball 命名格式：**
```
mysql-{version}-linux-glibc{glibc_version}-{arch}.tar.gz
```
示例：`mysql-8.0.41-linux-glibc2.28-x86_64.tar.gz`

**XtraBackup tarball 命名格式：**
```
percona-xtrabackup-{version}-linux-{arch}.tar.gz
```
示例：`percona-xtrabackup-8.0.36-30.1-linux-x86_64.tar.gz`

**arch 可选值：** `x86_64`、`aarch64`

## 3. 功能设计

### 3.1 新增命令：packages

```bash
# 查看当前 packages 目录状态
ops_db packages status

# 下载 MySQL 指定版本
ops_db packages download mysql --version=8.0.41 --os=ubuntu22

# 下载 XtraBackup 指定版本
ops_db packages download xtrabackup --version=8.0.36 --os=ubuntu22

# 自动下载当前系统对应的最新兼容版本
ops_db packages download --all

# 验证包完整性（MD5/SHA256）
ops_db packages verify
```

### 3.2 install 命令新增 --mirror / --offline 参数

```bash
# 模式一：使用内网镜像源安装（推荐）
ops_db install --mirror=intranet          # 使用环境变量配置的内网源
ops_db install --mirror=tencent           # 腾讯云镜像
ops_db install --mirror=aliyun           # 阿里云镜像

# 指定 MySQL 版本
ops_db install --mirror=intranet --version=8.0.41

# 远程安装（通过 SSH）
ops_db install --mirror=intranet --ssh-host=192.168.56.8 --ssh-password=123456

# 模式二：使用本地离线包安装（备选）
ops_db install --offline=/path/to/packages --version=8.0.41

# 同时安装 XtraBackup
ops_db install --mirror=intranet --with-xtrabackup
```

### 3.3 自动版本匹配逻辑

```
目标系统 (os_key)  →  推荐 MySQL 版本  →  manifest 中查找对应 tarball
                   →  推荐 XtraBackup 版本 → manifest 中查找对应 tarball
```

匹配规则：
1. 根据 `system_detect.py` 的 `MYSQL_VERSION_MAP` 确定目标 OS 应使用的 MySQL/XtraBackup 推荐版本
2. 在 manifest 中查找同版本或最接近的兼容版本
3. 若 manifest 中没有目标版本，报错退出并提示可用版本列表

## 4. 离线安装流程

### 4.1 Ubuntu/Debian 离线安装步骤

```
1. 解析 tarball → 解压到 /usr/local/mysql-{version}/
2. 创建 mysql 用户（若不存在）
3. 创建符号链接：ln -s /usr/local/mysql-{version} /usr/local/mysql
4. 初始化 data 目录：mysqld --initialize --datadir={datadir}
5. 生成 my.cnf（复用现有 write_my_cnf）
6. 设置 server-id、port、role 等参数
7. chown -R mysql:mysql {datadir}
8. systemctl start mysql
9. 验证连接（127.0.0.1:port, root, password）
```

### 4.2 CentOS/RHEL 离线安装步骤

同上，路径改为 `/opt/mysql-{version}/`。

### 4.3 XtraBackup 离线安装

```
1. 解析 tarball → 解压到 /usr/local/xtrabackup-{version}/
2. 创建符号链接：ln -s /usr/local/xtrabackup-{version} /usr/local/xtrabackup
3. 验证：xtrabackup --version
```

## 5. 与现有代码的集成

### 5.1 改动点

| 文件 | 改动内容 |
|------|---------|
| `ops_db/modules/install.py` | 新增 `_do_offline_install()` 分支；`install_mysql()` 增加 `offline` 参数 |
| `ops_db/__main__.py` | `install` 子命令增加 `--offline` 参数 |
| 新增 `ops_db/modules/packages.py` | packages 命令实现：download、status、verify |
| `ops_db/lib/system_detect.py` | 增加 `get_glibc_version()`（可选，用于兼容性检测）|

### 5.2 CLI 参数设计

**install 子命令参数扩展：**
```
--mirror TEXT       指定镜像源：intranet(环境变量) / tencent / aliyun / official（默认 tencent）
--offline TEXT      指定离线包目录路径（离线模式，与 --mirror 互斥）
--version TEXT      指定 MySQL 版本（离线/内网模式下必选或自动匹配）
--with-xtrabackup   同时安装 XtraBackup
```

**内建镜像源（可直接使用）：**

| 名称 | 类型 | 地址 |
|------|------|------|
| `tencent` | 腾讯云镜像 | mirrors.cloud.tencent.com |
| `aliyun` | 阿里云镜像 | mirrors.aliyun.com |
| `intranet` | 自定义内网源 | 环境变量 INTRANET_* 配置 |
| `internal` | 自定义内网源 | 需要用户先配置地址 |
| `official` | 官方源（海外，慢） | dev.mysql.com |

**packages 子命令参数：**
```
packages download [OPTIONS]
  --mysql VERSION      MySQL 版本（如 8.0.41）
  --xtrabackup VERSION XtraBackup 版本（如 8.0.36）
  --os TEXT            目标 OS（如 ubuntu22, centos7）
  --all                下载所有推荐版本

packages status [OPTIONS]
  --detail             显示详细信息

packages verify [OPTIONS]
  --fix                自动修复损坏的包
```

**config 子命令（管理内网源配置）：**
```
ops_db config set mirror.internal http://nexus.internal/repository/mysql/
ops_db config get mirror.internal
ops_db config list mirror
```

## 6. 错误处理

| 场景 | 处理方式 |
|------|---------|
| 内网源地址无法访问 | 报错：`无法访问内网源 {url}，请检查网络或更换镜像` |
| offline 目录不存在 | 报错：`离线包目录不存在: {path}` |
| manifest.json 缺失 | 报错：`离线包清单不存在，请运行 packages download` |
| 目标版本包不存在 | 报错：`packages 中未找到 MySQL {version}，可用版本: ...` |
| tarball MD5 不匹配 | 报错：`包 {name} 校验失败，请重新下载` |
| glibc 版本不兼容 | 报错：`{filename} 需要 glibc {required}，当前系统为 glibc {actual}` |
| 目标系统已安装 MySQL | 跳过安装，报错：`MySQL 已存在，请先卸载` |
| --mirror 和 --offline 同时指定 | 报错：`离线模式和内网源模式互斥，请只选择一种` |

## 7. 安全性考虑

1. **包完整性校验**：下载后计算 SHA256 并记录到 manifest（后续可扩展）
2. **禁止远程执行任意命令**：offline 模式下所有文件来自本地 tarball，不涉及远程下载
3. **路径限制**：仅允许读取 `packages/` 目录下的包，防止路径穿越

---

## 8. 向后兼容

- 若未指定 `--offline`，走现有在线安装流程（apt/yum/dnf）
- 现有 `install` 命令的所有参数保持不变
- 离线模式与在线模式互斥，不能同时使用

## 9. 用户使用流程示例

### 场景：内网服务器安装 MySQL 8.0

**步骤 1：在有网络的电脑上下载包**
```bash
# 在 56.7（管控机，有网络）上执行
cd /root/workspace/ops_db
ops_db packages download mysql --version=8.0.41 --os=ubuntu22
ops_db packages download xtrabackup --version=8.0.36 --os=ubuntu22
# 下载完成后，packages/ 目录包含所有 tarball
```

**步骤 2：将 packages 目录复制到目标机器**
```bash
# 56.7 和 56.8 在同一内网，可用 scp
scp -r /root/workspace/ops_db/packages root@192.168.56.8:/tmp/packages
```

**步骤 3：在目标机器上离线安装**
```bash
# 登入 56.8
ssh root@192.168.56.8
cd /root/workspace/ops_db
ops_db install --offline=/tmp/packages --version=8.0.41 --ssh-host=127.0.0.1
```

## 10. 后续扩展方向

1. **packages 同步**：支持从远程 HTTP 源同步包（`packages sync --source=http://nexus.internal`）
2. **自动下载依赖**：MySQL tarball 仍需 glibc 等基础库，可增加 `packages download --with-deps`
3. **Windows 支持**：MySQL 也有 Windows zip 包，可扩展支持 Windows 节点安装
4. **包版本自动更新**：定期检查新版 MySQL/XtraBackup 并提醒用户更新 packages
